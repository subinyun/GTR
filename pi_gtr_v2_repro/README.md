# GTR Reproducibility Package

This folder is a PI-facing guide for reviewing and rerunning the GTR experiments before paper submission. It does not modify the original experiment code. Instead, it gives a clean reading path, wrapper scripts, expected outputs, and result summaries.

## What To Review First

1. Read `docs/claim_map.md` for the seven-claim paper narrative.
2. Read `docs/results_summary.md` for the compact result numbers.
3. Read `docs/file_map.md` to see which source files are part of the final GTR path.
4. Read `docs/reproducibility.md` for the exact rerun order.
5. Compare rerun metrics against `results/expected_metrics.json`.

The paper story is organized around axis validity, coordinate calibration, hybrid superiority, interaction necessity, selective suppression, hard-negative margin improvement, and LLM routing improvement.

The main reproducible experiment path is CAIL2018 statute classification with GTR reranking, followed by LLM or LoRA decoding:

- GTR checkpoint: `output/cail2018_gtr_v2_only/full/hybrid_gtr_v2_best.pt`
- GTR report: `output/cail2018_gtr_v2_only/full/hybrid_gtr_v2_report.json`
- LLM prompts: `output/cail2018_gtr_v2_only/full/llm_prompts/`
- GPT-5.4 rerank results: `output/cail2018_gtr_v2_only/full/gpt54_gtr_v2_rerank_276/`
- Qwen3-8B LoRA results: `output/cail2018_gtr_v2_only/full/qwen3_8b_lora_sft/`

## Quick Sanity Check

From the repository root:

```bash
bash pi_gtr_v2_repro/scripts/00_check_environment.sh
```

This checks required data files, source scripts, optional API keys, and core Python packages.

## Minimal Reproduction

If the GTR checkpoint and prompt files are already present, rerun only the GPT-5.4 decoder/evaluator:

```bash
export OPENAI_API_KEY=...
bash pi_gtr_v2_repro/scripts/03_run_gpt54_rerank_eval.sh
```

For the most stable no-API reproducibility check, recompute metrics from the saved GPT-5.4 result file:

```bash
bash pi_gtr_v2_repro/scripts/08_recompute_saved_gpt_metrics.sh
```

Expected GPT-5.4 result on the 276-case sample:

- `raw_pool_gtr_rerank`: exact match `0.7428`, micro-F1 `0.7987`
- `raw_pool_gtr_rerank_score_prompt`: exact match `0.7609`, micro-F1 `0.8039`

## Full Reproduction Order

Run these from the repository root:

```bash
bash pi_gtr_v2_repro/scripts/00_check_environment.sh
bash pi_gtr_v2_repro/scripts/01_train_cail_gtr.sh
bash pi_gtr_v2_repro/scripts/02_build_llm_prompts.sh
export OPENAI_API_KEY=...
bash pi_gtr_v2_repro/scripts/03_run_gpt54_rerank_eval.sh
bash pi_gtr_v2_repro/scripts/05_build_qwen_lora_data.sh
bash pi_gtr_v2_repro/scripts/06_run_qwen_lora_smoke.sh
```

The public GitHub package does not include the full CAIL2018 split files or the BGE embedding cache because they are too large for a normal GitHub push. To retrain from scratch, place the CAIL2018 split files at `final_all_data/cail2018_statute_classification/`; the embedding cache is regenerated by the training script when missing.

For the full Qwen LoRA run, use:

```bash
bash pi_gtr_v2_repro/scripts/07_run_qwen_lora_full.sh
```

## Notes On Exact Reproducibility

The deterministic parts are the fixed sample seed, prompt construction, evaluation code, and metric computation. Small differences may still occur when retraining because of CUDA kernels, PyTorch version, GPU type, or upstream hosted LLM behavior. For paper audit, the most stable path is to preserve the generated prompts/results and rerun the evaluator.
